Photo Workflow Rules For This Folder

When working in D:\Code\HEIF-to-JPG:

1. Read README.md first.
2. If the user says "execute the script", "run the script", or asks to process a new batch, use the interactive workflow first:
   npm run workflow:photos
3. Before running, confirm or collect:
   - which config to use, default key.txt
   - JPEG quality
   - output suffix
   - input folder, default input
   - output folder, default output
   - OSS folder/prefix
   - whether GPS metadata should be removed
   - whether to upload immediately
   - public domain for returned links, default picture.nevergpdzy.cn
4. Prefer quality 90 by default unless the user explicitly asks for another value.
5. Source HEIC/HEIF photos belong in input/. Processed JPG files belong in output/.
6. Prefer reading local upload credentials from key.txt, then map them into OSS environment variables at runtime rather than writing secrets into tracked files. Default picture.nevergpdzy.cn binding uses bucket nevergpdzy-picture and region oss-cn-chengdu if key.txt does not specify them.
7. After upload, return the final public URLs if a public domain is available.
8. If the user wants public website images, remind them that keeping GPS metadata may expose location information.
